# 路线推荐功能说明

## 功能概述
该功能根据用户当前位置和目标终点，智能推荐最优的三条路线。系统会计算从当前位置到路线中任意站点的距离，选择总距离最短的路径，并优化途径点。

## 核心算法

### 最优路径计算
1. **遍历所有可能的起始站点**：对于每条路线，计算从当前位置到路线中每个站点的距离
2. **计算路线距离**：从每个可能的起始站点开始，计算到达终点的路线距离
3. **选择最短路径**：选择总距离（当前位置到起始站点 + 起始站点到终点）最短的路径
4. **优化途径点**：过滤掉距离过近的不必要途径点（距离小于100米的站点）

### 途径点优化规则
- 保留起始站点
- 保留终点站点
- 保留距离上一个站点超过100米的重要站点
- 过滤掉距离过近的小站点

## API接口

### 接口地址
```
POST /post/RouteController/queryRoutesNavigationLists
```

### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| latitude | Double | 是 | 当前纬度 |
| longitude | Double | 是 | 当前经度 |
| endId | String | 是 | 终点地点ID |
| schoolId | String | 是 | 学校ID |
| typeId | String | 否 | 路线类型（1:步行, 2:自行车, 3:公交） |

### 请求示例
```json
{
    "latitude": 39.9042,
    "longitude": 116.4074,
    "endId": "building_001",
    "schoolId": "school_001",
    "typeId": "1"
}
```

## 评分算法

### 评分因素
1. **距离评分（40%）**：总距离越短评分越高
2. **站点评分（30%）**：优化后的站点数量越少评分越高
3. **起点距离评分（20%）**：到最优起始站点距离越近评分越高
4. **路线类型评分（10%）**：步行>自行车>公交

### 计算公式
```
综合评分 = 距离评分×0.4 + 站点评分×0.3 + 起点距离评分×0.2 + 路线类型评分×0.1
```

## 推荐理由生成规则
- 排名第一：综合评分最高
- 起始站点位置：从路线起点开始 / 从路线中途站点开始
- 距离路线：距离路线很近（<0.3km） / 距离路线较近（<0.8km）
- 站点数量：站点较少，路径简洁（≤3个） / 站点适中（≤5个）
- 总距离：总距离很短（<1.5km） / 总距离较短（<3km）
- 路线类型：步行路线，健康环保 / 自行车路线，快速便捷 / 公交路线，舒适省力

## 算法优势

### 1. 智能路径选择
- 不是简单地选择路线起点，而是计算所有可能的起始点
- 选择真正的最短路径

### 2. 途径点优化
- 自动过滤掉不必要的近距离站点
- 保留关键导航点，简化路线

### 3. 多维度评分
- 综合考虑距离、站点数量、路线类型等因素
- 提供更合理的推荐排序

## 注意事项
1. 确保传入的经纬度格式正确
2. 终点ID必须在系统中存在
3. 路线类型参数可选，默认为步行路线
4. 返回的路线按综合评分从高到低排序
5. 途径点优化阈值可根据实际需求调整（当前为100米）
