## 1. 概述

会签节点监听器（MultiInstance Loop Listener）是 Flowable 工作流引擎中用于监听会签（多实例）节点执行情况的监听器。通过实现 `ExecutionListener` 接口，可以在会签节点的不同生命周期阶段执行自定义业务逻辑。

### 1.1 什么是会签节点

会签节点（Multi-Instance Task）是 BPMN 2.0 规范中的一种特殊任务节点，允许为同一个任务创建多个实例，通常用于：
- **并行会签**：多个审批人同时审批，所有人都通过才算通过
- **串行会签**：多个审批人按顺序审批，依次通过
- **投票表决**：多人投票，根据投票结果决定流程走向

### 1.2 监听器的作用

监听器可以在会签节点的以下时机执行：
- **start**：每个会签实例开始执行时
- **end**：每个会签实例执行结束时
- **take**：顺序流被触发时（较少使用）

## 2. 使用场景

### 2.1 典型应用场景

1. **进度监控**
   - 实时统计会签完成进度
   - 显示"已完成 3/5" 的进度信息
   - 在界面上展示会签状态

2. **业务通知**
   - 会签开始时发送通知给所有参与人
   - 会签完成时通知流程发起人
   - 会签超时时发送提醒

3. **数据统计**
   - 记录会签执行日志
   - 统计会签耗时
   - 分析会签通过率

4. **业务逻辑处理**
   - 所有会签实例完成后，自动计算投票结果
   - 根据会签结果更新业务状态
   - 触发后续业务流程

5. **异常处理**
   - 监控会签异常情况
   - 自动处理超时会签
   - 记录异常日志

## 3. 配置方法

### 3.1 在流程设计器中配置

#### 步骤 1：打开流程设计器
1. 登录系统，进入流程管理模块
2. 创建新流程或编辑已有流程
3. 打开流程设计器

#### 步骤 2：创建会签节点
1. 从左侧工具栏拖拽"用户任务"到画布
2. 选中该任务节点
3. 在右侧属性面板中找到"多实例类型"
4. 选择"并行"或"串行"
5. 配置会签参数：
   - **基数（Cardinality）**：会签实例数量，如 `5` 表示 5 人会签
   - **集合（Collection）**：会签人员集合，如 `${approvers}` 表示从流程变量获取
   - **元素变量（Element Variable）**：每个实例的变量名，如 `approver`
   - **完成条件（Completion Condition）**：会签完成条件，如 `${nrOfCompletedInstances/nrOfInstances >= 0.6}` 表示 60% 通过即可

#### 步骤 3：配置执行监听器
1. 在会签节点的属性面板中，找到"执行监听器"（Execution Listeners）
2. 点击"+"按钮添加监听器
3. 配置监听器参数：

| 参数 | 说明 | 示例值 |
|------|------|--------|
| **事件类型（Event）** | 选择触发时机 | `start` 或 `end` |
| **监听器类型（Listener Type）** | 选择监听器类型 | `Java class` |
| **类名（Class）** | 监听器完整类名 | `com.skyeye.activiti.listener.MultiInstanceloopListener` |

4. 如果需要监听多个事件，可以添加多个监听器：
   - 添加一个 `start` 事件的监听器（会签开始时触发）
   - 添加一个 `end` 事件的监听器（会签结束时触发）

#### 步骤 4：保存并发布
1. 保存流程模型
2. 发布流程定义
3. 测试流程，验证监听器是否正常工作

### 3.2 配置示例

#### 示例 1：监听会签开始事件
```
事件类型：start
监听器类型：Java class
类名：com.skyeye.activiti.listener.MultiInstanceloopListener
```

#### 示例 2：监听会签结束事件
```
事件类型：end
监听器类型：Java class
类名：com.skyeye.activiti.listener.MultiInstanceloopListener
```

#### 示例 3：同时监听开始和结束
可以添加两个监听器：
- 第一个：事件类型 `start`，类名 `com.skyeye.activiti.listener.MultiInstanceloopListener`
- 第二个：事件类型 `end`，类名 `com.skyeye.activiti.listener.MultiInstanceloopListener`

### 3.3 BPMN XML 配置（高级）

如果直接编辑 BPMN XML，监听器配置如下：

```xml
<userTask id="multiInstanceTask" name="会签审批">
  <multiInstanceLoopCharacteristics isSequential="false">
    <loopCardinality>5</loopCardinality>
    <!-- 或使用集合 -->
    <!-- <loopDataInputRef>approvers</loopDataInputRef> -->
  </multiInstanceLoopCharacteristics>
  <extensionElements>
    <flowable:executionListener 
        event="start" 
        class="com.skyeye.activiti.listener.MultiInstanceloopListener"/>
    <flowable:executionListener 
        event="end" 
        class="com.skyeye.activiti.listener.MultiInstanceloopListener"/>
  </extensionElements>
</userTask>
```

## 4. 代码实现

### 4.1 创建监听器类

创建一个实现 `ExecutionListener` 接口的 Java 类：

```java
package com.skyeye.activiti.listener;

import org.flowable.engine.delegate.DelegateExecution;
import org.flowable.engine.delegate.ExecutionListener;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class MultiInstanceloopListener implements ExecutionListener {
    
    private static final Logger LOGGER = LoggerFactory.getLogger(MultiInstanceloopListener.class);
    
    @Override
    public void notify(DelegateExecution execution) {
        // 在这里实现你的业务逻辑
    }
}
```

### 4.2 获取执行信息

在 `notify` 方法中，可以通过 `DelegateExecution` 对象获取各种信息：

```java
@Override
public void notify(DelegateExecution execution) {
    // 获取流程实例ID
    String processInstanceId = execution.getProcessInstanceId();
    
    // 获取当前执行实例ID
    String executionId = execution.getId();
    
    // 获取当前活动节点ID
    String activityId = execution.getCurrentActivityId();
    
    // 获取事件类型（start 或 end）
    String eventName = execution.getEventName();
    
    // 获取流程变量
    Object variable = execution.getVariable("变量名");
    
    // 设置流程变量
    execution.setVariable("变量名", "变量值");
}
```

### 4.3 获取会签相关信息

会签节点会自动设置以下流程变量：

```java
// 获取会签实例总数
Integer nrOfInstances = (Integer) execution.getVariable("nrOfInstances");

// 获取当前活跃的会签实例数量
Integer nrOfActiveInstances = (Integer) execution.getVariable("nrOfActiveInstances");

// 获取已完成的会签实例数量
Integer nrOfCompletedInstances = (Integer) execution.getVariable("nrOfCompletedInstances");

// 获取当前实例的循环计数器（从0开始）
Integer loopCounter = (Integer) execution.getVariable("loopCounter");
```

## 5. 会签相关变量说明

### 5.1 系统自动变量

Flowable 引擎在会签节点执行时，会自动设置以下变量：

| 变量名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| `nrOfInstances` | Integer | 会签实例总数 | `5` |
| `nrOfActiveInstances` | Integer | 当前活跃的实例数量 | `3` |
| `nrOfCompletedInstances` | Integer | 已完成的实例数量 | `2` |
| `loopCounter` | Integer | 当前实例的循环计数器（从0开始） | `0`, `1`, `2`... |
| `elementVariable` | Object | 元素变量值（如果配置了元素变量） | 当前会签人的用户ID |

### 5.2 变量使用示例

```java
// 计算完成进度
Integer total = (Integer) execution.getVariable("nrOfInstances");
Integer completed = (Integer) execution.getVariable("nrOfCompletedInstances");
double progress = (double) completed / total * 100;
LOGGER.info("会签完成进度: {}%", String.format("%.2f", progress));

// 判断是否全部完成
if (completed.equals(total)) {
    LOGGER.info("所有会签实例已完成！");
    // 执行后续业务逻辑
}

// 获取当前会签人（如果配置了元素变量）
String currentApprover = (String) execution.getVariable("approver");
LOGGER.info("当前会签人: {}", currentApprover);
```

## 6. 完整示例代码

### 6.1 基础示例

```java
package com.skyeye.activiti.listener;

import org.flowable.engine.delegate.DelegateExecution;
import org.flowable.engine.delegate.ExecutionListener;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class MultiInstanceloopListener implements ExecutionListener {

    private static final Logger LOGGER = LoggerFactory.getLogger(MultiInstanceloopListener.class);

    @Override
    public void notify(DelegateExecution execution) {
        try {
            // 获取基本信息
            String processInstanceId = execution.getProcessInstanceId();
            String executionId = execution.getId();
            String activityId = execution.getCurrentActivityId();
            String eventName = execution.getEventName();
            
            LOGGER.info("========== 会签节点监听器触发 ==========");
            LOGGER.info("流程实例ID: {}", processInstanceId);
            LOGGER.info("执行实例ID: {}", executionId);
            LOGGER.info("活动节点ID: {}", activityId);
            LOGGER.info("事件类型: {}", eventName);
            
            // 获取会签相关变量
            Object nrOfInstances = execution.getVariable("nrOfInstances");
            Object nrOfActiveInstances = execution.getVariable("nrOfActiveInstances");
            Object nrOfCompletedInstances = execution.getVariable("nrOfCompletedInstances");
            Object loopCounter = execution.getVariable("loopCounter");
            
            if (nrOfInstances != null) {
                LOGGER.info("会签实例总数: {}", nrOfInstances);
                LOGGER.info("活跃实例数: {}", nrOfActiveInstances);
                LOGGER.info("已完成实例数: {}", nrOfCompletedInstances);
                LOGGER.info("当前循环计数: {}", loopCounter);
                
                // 计算完成进度
                if (nrOfInstances instanceof Integer && nrOfCompletedInstances instanceof Integer) {
                    int total = (Integer) nrOfInstances;
                    int completed = (Integer) nrOfCompletedInstances;
                    if (total > 0) {
                        double progress = (double) completed / total * 100;
                        LOGGER.info("会签完成进度: {}%", String.format("%.2f", progress));
                        
                        // 如果全部完成，执行自定义业务逻辑
                        if (completed == total && "end".equals(eventName)) {
                            LOGGER.info("所有会签实例已完成！可以在这里执行后续业务逻辑");
                            // TODO: 在这里添加你的业务逻辑
                        }
                    }
                }
            }
            
            LOGGER.info("========== 会签节点监听器执行完成 ==========");
            
        } catch (Exception e) {
            LOGGER.error("会签节点监听器执行异常", e);
        }
    }
}
```

## 7. 常见问题

### 7.1 监听器不执行

**问题**：配置了监听器，但监听器方法没有被调用。

**可能原因**：
1. 监听器类名配置错误
2. 监听器类不在 classpath 中
3. 流程未正确发布
4. 事件类型配置错误

**解决方法**：
1. 检查类名是否正确，包括包名
2. 确认类已编译并打包到部署包中
3. 重新发布流程定义
4. 检查事件类型是否为 `start` 或 `end`

### 7.2 获取不到会签变量

**问题**：在监听器中获取 `nrOfInstances` 等变量返回 `null`。

**可能原因**：
1. 监听器在会签节点外部配置
2. 事件类型选择错误
3. 变量还未设置

**解决方法**：
1. 确保监听器配置在会签节点上，而不是父流程上
2. 使用 `start` 或 `end` 事件，而不是 `take` 事件
3. 在 `end` 事件中获取变量，此时变量已完全设置

### 7.3 监听器异常影响流程

**问题**：监听器抛出异常，导致流程执行失败。

**解决方法**：
```java
@Override
public void notify(DelegateExecution execution) {
    try {
        // 业务逻辑
    } catch (Exception e) {
        LOGGER.error("监听器执行异常", e);
        // 不抛出异常，避免影响流程执行
        // 或者根据业务需求决定是否抛出异常
    }
}
```

### 7.4 并行会签和串行会签的区别

**并行会签**：
- 所有实例同时创建和执行
- `nrOfActiveInstances` 在开始时等于 `nrOfInstances`
- 适合互不影响的审批场景

**串行会签**：
- 实例按顺序创建和执行
- `nrOfActiveInstances` 始终为 1（或 0）
- 适合需要按顺序审批的场景

## 8. 最佳实践

### 8.1 性能优化

1. **避免耗时操作**
   - 不要在监听器中执行数据库查询、外部服务调用等耗时操作
   - 如果必须执行，考虑使用异步方式

2. **合理使用事件类型**
   - 如果只需要在会签完成时执行逻辑，只配置 `end` 事件
   - 避免在 `start` 事件中执行复杂逻辑

3. **缓存数据**
   - 对于需要频繁访问的数据，考虑缓存
   - 避免重复查询数据库

### 8.2 异常处理

1. **捕获所有异常**
   ```java
   try {
       // 业务逻辑
   } catch (Exception e) {
       LOGGER.error("监听器执行异常", e);
       // 根据业务需求决定是否抛出异常
   }
   ```

2. **记录详细日志**
   - 记录关键信息，便于问题排查
   - 使用合适的日志级别

### 8.3 代码规范

1. **使用日志框架**
   - 使用 SLF4J + Logback，不要使用 `System.out.println`

2. **添加注释**
   - 为关键逻辑添加注释
   - 说明业务意图

3. **单一职责**
   - 监听器只负责监听和记录
   - 复杂业务逻辑应该抽取到 Service 层

## 9. 扩展开发

### 9.1 自定义监听器

除了基础的监听器，你还可以创建自定义监听器：

```java
public class CustomCountersignListener implements ExecutionListener {
    
    @Override
    public void notify(DelegateExecution execution) {
        // 自定义业务逻辑
        // 例如：调用外部系统、发送消息、更新状态等
    }
}
```

### 9.2 使用 Spring Bean

如果需要在监听器中使用 Spring Bean，可以通过工具类获取：

```java
import com.skyeye.common.util.SpringUtils;

public class CountersignListener implements ExecutionListener {
    
    @Override
    public void notify(DelegateExecution execution) {
        // 获取 Spring Bean
        NotificationService notificationService = SpringUtils.getBean(NotificationService.class);
        
        // 使用 Bean
        notificationService.sendNotification("会签完成");
    }
}
```

### 9.3 监听器参数配置

如果需要为监听器配置参数，可以使用表达式：

```java
public class CountersignListener implements ExecutionListener {
    
    private Expression notificationType;
    
    @Override
    public void notify(DelegateExecution execution) {
        String type = (String) notificationType.getValue(execution);
        // 根据参数执行不同逻辑
    }
}
```

在流程设计器中配置参数：
- 监听器类型：`Java class`
- 类名：`com.skyeye.activiti.listener.CountersignListener`
- 字段（Fields）：
  - 名称：`notificationType`
  - 类型：`string`
  - 值：`email` 或 `${notificationType}`（从流程变量获取）
